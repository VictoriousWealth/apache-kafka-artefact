# Sweep Framework

## Core Model

The artefact is structured around three concepts:

- `baseline`
- `sweep`
- `run`

## Baseline

A baseline is the fixed reference configuration for a family of benchmark runs.

It includes values such as:

- `security_mode`
- `broker_count`
- `partition_count`
- `replication_factor`
- `message_size_bytes`
- `num_records`
- `target_messages_per_second`
- `batch_size`
- `linger_ms`
- `acks`

## Sweep

A sweep definition specifies:

- the baseline it extends
- the single variable to vary
- the range of values to test
- the number of repeated trials per value, if required

Examples:

- `security_mode = [plaintext, tls, mtls]`
- `message_size_bytes = [1024, 10240, 102400]`
- `target_messages_per_second = [1000, 10000, 50000]`

## Run

A run is one concrete execution generated by applying one sweep value to a baseline.

Example:

- baseline: `plaintext-default`
- sweep: `message_size_bytes`
- value: `10240`

This produces one resolved run where all baseline values remain fixed except `message_size_bytes`.

If a sweep defines `trials`, multiple resolved runs are generated for the same sweep value so the framework can later aggregate mean, minimum, and maximum metrics across repeated trials.

## Current Implementation Boundary

The current execution path supports plaintext producer performance runs over the sweep framework.

TLS and mTLS will later reuse the same baseline and sweep model rather than introducing a separate execution model.
